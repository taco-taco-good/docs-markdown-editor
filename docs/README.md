# Foldmark Docs

This folder only keeps documents that describe the current product and codebase.

Old planning-heavy documents were removed because they no longer matched the current implementation.

## Current documents

- [DESIGN.md](./DESIGN.md)
  Product UI primitives, token rules, and interaction consistency guidelines.

- [EDITOR-ARCHITECTURE.md](./EDITOR-ARCHITECTURE.md)
  How the current CodeMirror-based markdown editor works, including session state, live preview, tabs, and save flow.

- [GIT-WORKFLOW.md](./GIT-WORKFLOW.md)
  Repository branch and merge conventions used for Foldmark.

## What is intentionally not here

- long-form speculative PRDs
- outdated Tiptap-based editor plans
- future architecture documents that do not match the current code

If a document stops matching the code, it should be rewritten or removed instead of being kept as historical clutter.
